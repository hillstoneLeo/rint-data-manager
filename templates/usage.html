使用说明

系统概述

RINT 数据分享平台将下面两种模式整合在了一起：

  • 通过 Web 页面手工管理的数据文件：通过浏览器界面进行数据上传和下载
  • Git + DVC 管理的数据文件：通过 DVC 命令行（或 VS Code 插件）进行数据版本控制和同步

准备工作

  1. Web 页面通道：注册并登录账户
  2. DVC 通道：配置 DVC 远程存储 
    
dvc remote add  http://...
dvc remote modify  auth basic
dvc remote modify  user 
dvc remote modify  password

Web 页面通道

登录系统后，在数据仓库页面按照提示上传/下载数据（包括文件和文件夹）。

DVC 通道

下载数据

git clone <your-project>
dvc pull

上传数据

Git + DVC 上传数据工作流：

# 添加数据到 DVC
dvc add <data-file-or-folder>

# 提交更改到 Git
git add <your-data-file.dvc>
git commit -m "Add data via DVC"

# 推送数据到远程存储
dvc push
git push

常见问题

Q: 我的系统没安装 DVC？

A: 通知系统管理员安装 DVC，也可以在用户层级用 pipx, mise 等工具安装 DVC。

Q: 怎样在代码库里使用别人分享的数据？

A: 下载这个数据的 .dvc 文件到代码库里，然后 `dvc push`。

Q: Winodws 系统可以使用 DVC 吗？

A: 可以，下载安装包安装即可，目前还没实现 Winodws 上的自动元数据解析，上传数据的名称可能不对，后续版本会提供支持。

Q: 我能手工修改数据项的元数据吗？

A: 目前还不能，后续版本实现。
