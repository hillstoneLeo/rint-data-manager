使用说明

系统概述

RINT 数据分享平台将下面两种使用和分享数据的方式整合在了一起：

  • 通过 Web 页面手工管理的数据文件：通过浏览器界面进行数据上传和下载
  • Git + DVC 管理的数据文件：通过 DVC 命令行（或 VS Code 插件）进行数据版本控制和同步

准备工作

在首页注册并登录账户就可以通过 Web 页面上传、下载数据文件啦。

如果需要在源码项目中使用 DVC 管理数据文件，请创建文件 ~/.config/dvc/config: 

[core]
remote = myremote
['remote "myremote"']
    url = {{ server_url }}/dvc
    auth = basic
    user = <registered-email>
    password = <registered-password>
其中 <registered-email> 和 <registered-password> 是前面注册的 Email 和密码。

在 Windows 系统上，这个配置文件的位置为：%LocalAppData%\iterative\dvc\config。

这里的注册、配置都是一次性工作，不需要为每个项目单独设置。

Web 页面通道

登录系统后，在数据仓库页面按照提示上传/下载数据（包括文件和文件夹）。

DVC 通道

下载数据

git clone <your-project>
dvc pull

上传数据

Git + DVC 上传数据工作流：

# 添加数据到 DVC
dvc add <data-file-or-folder>

# 提交更改到 Git
git add <your-data-file.dvc>
git commit -m "Add data via DVC"

# 推送数据到远程存储
dvc push
git push

常见问题

Q: 我使用的 Linux 服务器没安装 DVC？

A: 通知系统管理员安装，也可以在用户级用 pipx, mise 等工具安装 DVC。

Q: Winodws 系统可以使用 DVC 吗？

A: 可以，下载安装包安装即可。

Q: 怎样在代码库里使用别人分享的数据？

A: 最常见的情况是数据对应的 .dvc 文件已经在代码库里，只要 clone 代码库然后执行 `dvc pull`，所有数据就下载到正确的位置上了。

如果要用的数据还没有对应的 .dvc 文件，可以从 Web 页面上下载这个 .dvc 文件到代码库里，然后 `dvc pull`。

Q: 收集的数据文件想上传到平台上，用 Web 还是 DVC 方式好？

A: 推荐使用 Web 界面上传，完整填写各项字段，方便自己和他人使用。

Q: 我能通过 DVC 上传、下载文件，但在 Web 界面显示的文件名不正确。

A: 通知管理员检查元数据收集模块。

Q: 已上传数据各项特征值还能修改吗？

A: 目前还不能，后续版本会增加特征编辑功能。

Q: 上传数据窗口里面的 父级项目 一栏是做什么用的？

A: 这项用来构建数据文件之间的关系，以构建数据谱系图，目前忽略它即可。
